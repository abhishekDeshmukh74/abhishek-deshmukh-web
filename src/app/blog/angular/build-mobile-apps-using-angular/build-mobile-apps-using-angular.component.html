There are a few options available for creating mobile apps with Angular. In this post, you will explore various ways you can create mobile applications for Android and iOS using Angular including their pros and cons. So without further ado, let's get started

Why though?

When you need to build both a web and a native mobile app, you have to create, develop, and maintain two separate projects. Although this got the job done it required more time and resources to achieve it. It is quickly becoming apparent that we can do better than that.

Now in this era where we need to develop something for both a web and a native mobile app with fewer efforts, money, and time, there are a few options available for creating a mobile app with Angular

1: Progressive web app (PWA)

For those who don't know what a PWA is - A Progressive Web App (PWA) is an app that uses modern Web capabilities to offer users a very similar (if not better) experience to a native app

I'll create a new Angular app and will follow the following steps to convert your Angular app into a PWA

  • ng new angular-app-pwa
  • cd angular-app-pwa
  • ng add @angular/pwa
  • npm i -g http-server
  • http-server -p 8080 -c-1 dist/angular-app-pwa

The above command completes the following actions

  • Adds the @angular/service-worker package to your project
  • Enables service worker build support in the CLI
  • Imports and registers the service worker in the app module
  • Updates the index.html file with a link to add the manifest.json file and adds meta tags for theme-color
  • Installs icon files to support the installed Progressive Web App (PWA)
  • Creates the service worker configuration file called ngsw-config.json, which specifies the caching behaviors and other settings

This a repo following the above steps on Github and you can see a working demo here

If you go there from mobile, a banner popup like this will appear. After clicking it, our app will be installed on your phone

Pros and Cons

  • They behave with the user as if they were native apps, in terms of interaction and navigation
  • Information is always up-to-date thanks to the data update process offered by service workers
  • Exposed over HTTPS protocol
  • Works Offline
  • Easily shared via URL without complex installations
  • Higher device battery drain
  • PWA support only started since Angular 6
  • not all devices support the full range of PWA features
  • Not a real native app, just a browser view in the app so hardware are not utilized well

2: Cordova

This is the easiest and least time-consuming way of taking an existing Angular application that you have and ship it as a mobile application on mobile devices using Cordova

To install Cordova run this command - npm install -g cordova

I have also installed android SDK because I am going to point to an Android app if you want to deploy as an ios application, you need to have Xcode and other tools

I will jump back to the terminal. I already have an Angular project so I am assuming you'll be taking an existing project as well. If not it's still easy to do the same procedure if you're creating a new project as well

  • cordova create angular-cordova-app
  • cd angular-cordova-app
  • cordova platform add android
  • Create a prod build of an Angular app using - ng build --prod
  • Goto dist folder. Open index.html file. Change to because when I serve the application from the base or root of the domain. We are serving from an existing file system of the operating system, in our case it's Android
  • Paste it in angular-cordova-app/www folder
  • cordova run android
  • Have a cup of coffee till the build process finishes

Pros and Cons

  • Very Quickly and effortlessly a mobile app can be created
  • Not a real native app, just a browser view in the app so hardware are not utilized well
  • Sometimes, the App Store will reject this app as our app is just a browser view running inside an app window

3: NativeScript

As mentioned on NativeScript website If you know Angular, you're already ready to develop amazing native mobile apps with NativeScript. With NativeScript and Angular, a single code base and skillset can be used to create web apps and native mobile apps with 100% native performance and power. Buttery smooth animations, direct access to 100% of the native platform APIs, itâ€™s pretty amazing.

Pros and Cons

  • Code and Skill Reuse, You can reuse JavaScript npm modules, as well as native iOS CocoaPod and Android Arsenal libraries, directly in NativeScript
  • True Native Mobile app
  • Cross-platform. Builds for Android and iOS. It deals with the platforms differences very well
  • Solved problem when you can't afford separate teams for Android and iOS
  • Backed up by Google and Telerik
  • Nativescript runs everything on a single thread. i.e., the UI thread. If you want to offload some processing, you have to use web workers
  • The final size for Android could be huge compared with hybrid tools or the same NativeScript size for iOS
  • The Styling and CSS is currently very limited
